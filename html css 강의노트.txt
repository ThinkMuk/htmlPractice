CSS Display------------------------------------------------------------------------------------------------------------
대표 종류 4가지

block: 박스를 쌓듯이 위에서 아래로 내려오는 상자를 만듦 (박스가 위에서 아래로 내려오는 효과를 띔)
Inline: 우측방향으로 쌓임 (속성을 적용시켜놓아도 텍스트 요소를 네모로 감싸주는 효과만 띄어줌) / width height의 효과를 받지 않음
Inline-block: 기본적으로는 우측으로 나열하는 속성을 띄지만, width height를 온전히 적용시킴 (박스가 왼쪽부터 쌓이는 효과를 보이게 됨)
None: 삭제시킴

Position------------------------------------------------------------------------------------------------------------
총 4가지+a가 존재함

Static: 기본값
Relative: 자기 자신을 기준으로
Absolute: 부모 요소를 기준으로
Fixed 뷰포트를 기준으로

Sticky: 스티키 스크롤

Cascading------------------------------------------------------------------------------------------------------------
덮여씌워지는 순서는
inline stylesheet >ID style > class style > Tag 순서임 

기초적인 Box Model:

크기 순서대로:
	Margin > Border > Padding > content
Border 까지의 라인이 Content height 와 Content width, 즉 컨텐츠들을 보여주는 너비와 높이이다.

반응형을 위한 css 단위 이해하기------------------------------------------------------------------------------------------------------------

% = 넓이의 몇퍼센트인지 표시하는 단위
* max-width= 말 그대로 최대폯을 의미함. 퍼센트 외에 크기가 얼마나 커질 수 있는지 한계점을 나타내는 함수.
em = '상위' 요소의 폰트 사이즈의 몇 배인가 나타내는 단위
rem = '최상위' 요소의 폰트 사이즈의 몇 배인가 나타내는 단위
vw = view port width 의 약자 (전체 화면을 얘기함) (퍼센트로 같이 줄어듦) ex) width: 20vw (뷰포트 기준으로 20퍼센트를 차지하는 것)
vh = view port height 의 약자. 위에 view port width 와 같은 개념을 가짐

SEO / 메타태그------------------------------------------------------------------------------------------------------------
SEO는 구글에서 검색할때 노출하는걸 의미함

index(색인): 글을 등록하면 색인에 저장을 함
crawling(크롤링) : 데이터를 웹에서 추출하는것을 의미함
web crawler (크롤러): 크롤링하는 소프트웨어를 의미함
Google bot(구글 봇): 구글에서 만든 크롤러 봇

*요약글은 160자, 한글은 80~110자가 가장 적당함 (검색엔진에 영향을 미치지는 않지만, 검색을 했을때 사용자들이 자기가 필요한 정보인지 직접적으로
전달을 할 수 있다.)

* 페이지 썸네일 이미지 주소는 1200x630을 권장함.
(작으면 공유할때 1:1로 나오게 되어 주목도가 떨어지게 됨)

* html을 시멘틱 구조로 작성하는것도 아주 중요함
(구글 크롬에 확장프로그램:headings map을 사용하면 훨씬 직접적으로 알아보기 쉬움)
head에 잘 적어둬야 노출이 잘 됨
